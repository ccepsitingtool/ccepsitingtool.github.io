More Information on the Purpose and Creation of the Siting Tool

Suitable Sites

The entirety of the “suitable sites” - .5 mile grid cells covering each county were created using a combined approach for road density and “points of interest” density. A grid cell needed to either contain a sufficient density of roads or at least two points of interest. Points of interest were defined by the research team to be both governmental and non-governmental buildings (reflective as two different layers in the tool) that could serve as potential polling sites.

Facility Location

The suggest sites layer provides one view of suggested locations for polling sites. This layer is created using a facility location model (the basic code framework is here, and here is an interactive demo.) Suitable site grid cells were grouped using a k-means cluster analysis where sites were clustered on the registered voter population. The facility location model was run on these clusters, with a cost function being constructed based on commute time. Clusters were selected for site inclusion, and within each cluster the highest diversity scoring site was recommended for the location of the polling site.

Scoring model

Following a long tradition of scoring for potential siting locations (from fields such as construction, emergency management and others) all suitable sites were scored for their performance on the SB 450 metrics.
